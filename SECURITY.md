# Security Policy

## 支援版本

| 版本 | 支援狀態 |
| --- | --- |
| 0.2.x | ✅ 支援中 |
| < 0.2 | ❌ 不再支援 |

## 回報安全漏洞

如果您發現安全漏洞，請**不要**透過公開的 GitHub Issues 回報。

請透過以下方式私下回報：

1. **Email**: [您的安全回報 email]
2. **GitHub Security Advisories**: 使用 GitHub 的私人安全回報功能

### 回報內容

請在回報中包含：

- 漏洞描述
- 重現步驟
- 潛在影響
- 建議的修復方式（如果有的話）

### 回應時間

- 我們會在 **48 小時內** 確認收到您的回報
- 我們會在 **7 天內** 提供初步評估
- 我們會盡快發布修復版本

## 安全最佳實踐

### 內容安全策略 (CSP)

本應用程式實作了嚴格的 CSP：

- 只允許從 `'self'` 載入腳本
- 限制圖片來源為已知的安全來源
- 限制網路連線至必要的 YouTube 相關域名

### 依賴安全

- 我們使用 Dependabot 自動更新依賴
- 所有依賴在合併前都會經過審查
- 定期進行安全審計

### 資料處理

- 所有使用者資料都儲存在本地
- 不會傳輸個人資料到外部伺服器
- 歷史記錄等敏感資料可由使用者隨時刪除

## 已知安全考量

1. **外部命令執行**: 本應用程式使用 yt-dlp 和 ffmpeg 作為外部工具。請確保這些工具來自可信來源。

2. **網路請求**: 應用程式會向 YouTube 發送請求以取得影片資訊。這些請求遵循 YouTube 的使用條款。

## 貢獻安全改進

歡迎提交安全相關的 PR！請確保：

1. 清楚說明改進的安全問題
2. 提供測試以驗證修復
3. 不要破壞現有功能
